## HYP-002 Blocking 验证

scenario #7 PASS = HYP-002（"manifest 模型足以支撑幂等 + 卸载，且不会误删宿主自加 skill"）通过。
具体做法：install opencode copy → 在 `.opencode/skills/my-own-skill/SKILL.md` 放用户内容 → uninstall →
- HF 25 个 skill 子目录全部 `rm -rf`（per-skill manifest entries）
- `my-own-skill/SKILL.md` 仍存在（不在 manifest entries 中）
- `.opencode/skills/` 父 dir 因仍含 my-own-skill 自然非空，`rmdir` 静默失败 → 保留
- manifest + readme 被删
hf-completion-gate 阻塞条件已满足。

## NFR-002 验证

scenario #12 PASS = NFR-002（"install 中途失败可恢复，宿主仓库回到 install 前状态"）通过。
具体做法：把 host 目录设为只读 → 跑 install → 第一个 mkdir 失败 →
- `set -Eeuo pipefail` 让 ERR trap 跨函数继承
- `mark_will_create dir "$HOST/.opencode" ""` 已预登记 → trap 触发 rollback
- rollback 用 `rm -rf` 清除；INSTALLED 反向迭代
- manifest + readme 不落盘
- 宿主 `find` count = install 前 count（assert_eq PASS）

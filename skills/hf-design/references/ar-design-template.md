# 工作项级设计说明书（design.md）

使用说明：

- 本模板用于 `hf-design` 产出组件根下的工作项级设计 `features/<id>-<slug>/design.md`（或团队覆盖路径），并在 `hf-ship` 收尾时 promote 到组件根下 `docs/ar-designs/AR<id>-<slug>.md`（或团队覆盖路径）。
- 正式交付件必须删除模板说明、示例业务内容和任何占位符；不得残留 `AI提示`、`TBD`、`{DATE}`、变量替换规则等模板痕迹。
- 测试设计必须是本文档章节，不得拆成独立 `test-design.md`。
- 本模板采用工作项级设计作业结构：AR 概述、动态行为、功能点分解、实现设计、适用领域场景设计、重构设计、测试设计、高质量设计增补、模板修订记录均为必填骨架。领域内容只在命中对应领域技能 description 时填写；未命中的领域写明 N/A 与判定依据。
- 第 7 章「高质量设计增补」承载 `hf-design` 的核心质量判断（接口契约六项、错误模型、数据所有权、简单性检验），填写判据见 `hf-design/SKILL.md` 对应章节；微小修改可按 `using-hf` §6 裁剪规则精简其他章节，但第 6、7 章不可省。

## 7. 高质量设计增补（必要）

> 本章承载第三层内在质量的设计判断，是评审的重点章节。填写判据与正反例见 `hf-design/SKILL.md`。第 4 章描述"结构长什么样"，本章回答"这个结构为什么是对的、契约是否完整、失败时会发生什么"。

### 7.1 接口契约细化（六项）

对第 4.4 章接口表中每个**新增或语义变更**的接口，逐个写全六项契约（可冷读标准：拿着契约不看实现就能写出测试）：

#### <接口名>

| 契约项 | 内容 |
|---|---|
| 输入与前置条件 | 参数含义、单位、合法范围、NULL 语义、调用上下文限制（可否在中断里调） |
| 输出与后置条件 | 返回值、出参、成功后系统状态的变化 |
| 错误语义 | 每个错误码的触发条件；**出错后系统状态如何**（见 7.2） |
| 副作用 | 写了什么状态、发了什么事件、持有了什么资源 |
| 并发与时序 | 线程安全性、可重入性、阻塞行为、超时 |
| 兼容性 | modify/remove 时：旧调用方迁移策略、错误码集变化、废弃计划 |

### 7.2 错误模型

| 设计项 | 内容 |
|---|---|
| 错误分类与策略 | 本工作项涉及哪些类别（调用方编程错误 / 可预期运行时失败 / 环境硬件故障 / 不可恢复内部矛盾），各自怎么处理 |
| 传播策略 | 错误在哪层翻译、哪层处理；默认在模块边界翻译一次，中间层只透传 |
| 失败路径状态保证 | 每个可失败操作失败后：已发生的副作用是回滚、保留还是明确的中间态。「出错后状态未定义」按 critical 处理 |

# 规格评审 Checklist

对照 `spec.md` 逐条检查。加载的 `ext-*` 扩展若声明了规格阶段的追加检查项,一并检查。

## 可测性

- [ ] 每条 FR 描述的是可观察的行为,而非实现方式或愿望
- [ ] 每条 FR 至少有一条 Given/When/Then 验收标准,且结果可判定(第三方能明确说出通过/不通过)
- [ ] 每条 NFR 有量化阈值与验证方式;无"足够快""稳定""良好体验"类措辞
- [ ] 验收标准覆盖了失败路径与边界值,不只有 happy path

## 完整与一致

- [ ] 背景说清了为谁解决什么问题;成功标准可判定
- [ ] 范围内/范围外显式列出,与 FR 清单一致(没有 FR 落在范围外,也没有范围内能力缺 FR)
- [ ] 需求之间无冲突、无重复
- [ ] 约束注明来源;假设写明失效影响
- [ ] 阻塞性开放问题已全部解决;残留的均已标注"非阻塞"

## 边界纪律

- [ ] 无设计泄漏:没有类名、接口签名、表结构、框架选型等 HOW 层内容
- [ ] 无未经用户确认却写成事实的假设
- [ ] 每条需求粒度适中:一条 FR 只描述一个行为,复合需求已拆分

# 3-Client × 3-Layer AGENTS.md Templates

> 三客户端各自的 root / mid / leaf 三层模板，互不依赖。架构师选定客户端后，本 skill 按此填充骨架。

## Layer 定义

| Layer | 触发条件 | 内容侧重 |
|---|---|---|
| **root** | 项目根目录 | 项目总览 / 高层架构 / 团队约定 / 主入口 |
| **mid** | 含 ≥ 2 子目录或含 README.md 的中间目录（src/ / packages/ / examples/ 等） | 该子模块的职责 / 内部约定 / 与邻接模块的边界 |
| **leaf** | 仅含源文件无子目录（src/components/ / src/hooks/ 等） | 该叶子目录的具体 pattern / convention / anti-pattern |

---

## OpenCode 模板（`AGENTS.md`）

OpenCode 的 `directoryAgentsInjector` hook 会按目录层级自动注入相应文件。文件名固定为 `AGENTS.md`，无 frontmatter 要求；但建议加 `> generated by hf-context-mesh, fill in conventions yourself` 的 disclaimer 行。

### root template

```markdown
# AGENTS.md — Project Root

> Generated skeleton; architect to fill in conventions.

## Project Purpose

<一句话项目目标>

## Top-level Architecture

<最少必要的高层视图：3-5 个模块 + 1-2 个关键交互>

## Conventions

- <代码风格 / 命名 / 测试纪律 / commit 规范 ...>

## Typical Patterns

- <项目内最常用的 N 个 pattern + 一行说明>

## Anti-Patterns

- <项目内禁止的 N 个 pattern + 简要理由>

## Cross-Skill Notes

- <与 vendoring 进来的 skill pack 的协作约定 / 特殊豁免>
```

### mid template

```markdown
# AGENTS.md — <module-name>

> Generated skeleton; architect to fill in conventions.

## Module Purpose

<该子模块的职责，1-2 句>

## Boundary

- 上游：<谁调用本模块>
- 下游：<本模块调用谁>
- 不应跨边界做：<反例>

## Conventions

- <该子模块特有的命名 / 错误处理 / 日志 / ...>

## Typical Patterns

## Anti-Patterns
```

### leaf template

```markdown
# AGENTS.md — <leaf-dir-name>

> Generated skeleton; architect to fill in conventions.

## Leaf Purpose

<本目录承载哪一类源文件>

## Conventions

- <最具体的 file 内 pattern / 命名 / 单元测试位置 / ...>

## Typical Patterns

## Anti-Patterns
```

## Cursor 模板（`.cursor/rules/<dir-slug>.mdc`）

Cursor 用 `.mdc` 文件 + frontmatter `globs:` 实现按目录触发；所有 .mdc 都放 `.cursor/rules/` 下，文件名按目录 slug 区分（如 `src-components.mdc`）。

### root template

```markdown
---
description: Project root context for AI agents
globs:
  - "**/*"
alwaysApply: false
---

# Project Root Context

> Generated skeleton; architect to fill in conventions.

## Project Purpose
## Top-level Architecture
## Conventions
## Typical Patterns
## Anti-Patterns
## Cross-Skill Notes
```

### mid template

```markdown
---
description: Context for <module-name>
globs:
  - "<module-path-from-root>/**"
alwaysApply: false
---

# <module-name> Context

> Generated skeleton; architect to fill in conventions.

## Module Purpose
## Boundary
## Conventions
## Typical Patterns
## Anti-Patterns
```

### leaf template

```markdown
---
description: Context for <leaf-dir-name>
globs:
  - "<leaf-path-from-root>/**"
alwaysApply: false
---

# <leaf-dir-name> Context

> Generated skeleton; architect to fill in conventions.

## Leaf Purpose
## Conventions
## Typical Patterns
## Anti-Patterns
```

注意：Cursor 全部 `.mdc` 集中 `.cursor/rules/`，**不**散落到各目录（与 OpenCode / Claude Code 不同）。`globs` 字段决定每条 rule 在哪些目录被触发。

## Claude Code 模板（`CLAUDE.md`）

Claude Code 的 `CLAUDE.md` 文件命名约定与 OpenCode `AGENTS.md` 平级，但内容由 Claude Code 客户端自身的 plugin / `.claude/` 加载机制注入。同目录可与 OpenCode `AGENTS.md` 共存（OpenCode 不读 CLAUDE.md，反之亦然）。

### root template

```markdown
# CLAUDE.md — Project Root

> Generated skeleton; architect to fill in conventions.

## Project Purpose
## Top-level Architecture
## Conventions
## Typical Patterns
## Anti-Patterns
## Cross-Skill Notes
```

### mid template

```markdown
# CLAUDE.md — <module-name>

> Generated skeleton; architect to fill in conventions.

## Module Purpose

## Anti-Patterns
```

### leaf template

# CLAUDE.md — <leaf-dir-name>

## Anti-Patterns

## 三客户端共存场景

如果架构师选 `all 3`，同一目录下会出现 `AGENTS.md` + `CLAUDE.md`（两个文件），并在 `.cursor/rules/<dir-slug>.mdc` 出现一份 Cursor rule。

为避免架构师为同一目录写 3 份内容：建议在 `AGENTS.md` 写主版本，`CLAUDE.md` 与 `.mdc` 仅写 `> 见 ./AGENTS.md`。或反之以 `CLAUDE.md` 为主。具体由架构师拍板（本 skill 不替决）。

---

## 占位填充的最小集

骨架生成时，每个 Conventions / Typical Patterns / Anti-Patterns 段下放一行 placeholder：

```markdown
## Conventions

- <fill in: e.g. all functions must have a doc comment; tests live in tests/ next to source>
```

让架构师扫一眼就知道要填什么；本 skill **不预填具体内容**。

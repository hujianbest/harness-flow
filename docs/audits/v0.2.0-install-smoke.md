# v0.2.0 Install Smoke (placeholder)

> **Status**: placeholder. ADR-002 D3 marks "real-environment install smoke" as a v0.2.0 GA hard gate. This document is the **target shape** of the smoke report; actual results must be filled in by a maintainer running each client in its real environment, since these checks cannot be performed inside the repository sandbox.

## Scope (ADR-002 D3)

This smoke covers every officially supported client in v0.2.0:

1. **Claude Code** marketplace install: `/plugin marketplace add hujianbest/harness-flow` + `/plugin install harness-flow@harness-flow`; verify all 6 slash commands are loaded.
2. **OpenCode**: clone-and-open topology + `/skills` listing all 24 skills (22 `hf-*` + `using-hf-workflow` + new `hf-browser-testing`).
3. **Cursor**: open the repo; verify `.cursor/rules/harness-flow.mdc` is loaded with `alwaysApply: true`; run the recommended first prompt.
4. **Gemini CLI**: `gemini skills install ./skills/`; verify `gemini skills list` shows all 24 skills; verify `/hf` `/spec` `/planning` `/build` `/review` `/ship` are recognized.
5. **Windsurf**: open the repo; verify `.windsurf/rules.md` is auto-loaded; run the recommended first prompt.
6. **GitHub Copilot**: open the repo in a Copilot-enabled IDE; verify `.github/copilot-instructions.md` is auto-loaded; run the recommended first prompt.
7. **Kiro**: open the repo; verify `.kiro/skills` symlink resolves; run Kiro's skill-listing equivalent and confirm 24 entries.

Acceptance rule: GA requires every client to be `GA-acceptable`. Any `blocker` → stop GA, file issue, fix, re-run that client.

## Why this lives in the repo as a placeholder

Per `CONTRIBUTING.md` "Known Limitations" (carried from v0.1.x stabilization), these checks **cannot** be performed inside the repository sandbox — they require:

- A real Claude Code install with marketplace access.
- A real OpenCode install with `skill` tool.
- Real Cursor / Windsurf / Copilot / Kiro / Gemini CLI binaries.

Shipping the placeholder allows the v0.2.0 release branch to make ADR-002 D3 visible and traceable; the actual results land via a follow-up commit by whoever runs the smoke in the real environments.
